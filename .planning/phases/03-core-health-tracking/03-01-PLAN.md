Exercise model (existing — DO NOT modify):
  category enum (English):  'yoga' | 'cardio' | 'weights' | 'stretching'
  difficulty enum (English): 'easy' | 'medium' | 'hard'
  steps: Array<{ order: number; instruction: string; durationSeconds?: number }>
  imageUrl: string | null
  Index: { category: 1, isActive: 1 }

GET /api/exercises
  Headers: Authorization: Bearer <accessToken>
  Query: ?category=yoga|cardio|weights|stretching (optional)
  Success 200: { success: true, data: IExercise[] }
  Errors:
    400 — invalid category enum → "Danh mục không hợp lệ"
    401 — missing/invalid token → "Token không hợp lệ hoặc đã hết hạn"

GET /api/exercises/:id
  Headers: Authorization: Bearer <accessToken>
  Path param: id (Mongo ObjectId-like 24-hex string)
  Success 200: { success: true, data: IExercise }
  Errors:
    400 — id is not a 24-hex string → "ID bài tập không hợp lệ"
    404 — id is valid ObjectId but no document → "Bài tập không tồn tại"
    401 — auth → "Token không hợp lệ hoặc đã hết hạn"

Seed contract:
  - Run via `npm run seed` from `backend/` cwd.
  - Reads MONGODB_URI from process.env (uses backend/.env via dotenv if not already loaded).
  - 25 yoga + 25 cardio + 25 weights + 25 stretching = 100 documents.
  - Vietnamese name + Vietnamese description + 3 Vietnamese steps minimum (D-41).
  - imageUrl: null on every row (D-42).
  - Idempotent: if Exercise.countDocuments({}) >= 100, skip insert.
