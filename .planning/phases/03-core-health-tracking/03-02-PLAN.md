    ```
    const d = new Date(input);
    const utc7Ms = d.getTime() + 7 * 60 * 60 * 1000;
    const dayStartUtc7 = Math.floor(utc7Ms / 86400000) * 86400000;
    return new Date(dayStartUtc7 - 7 * 60 * 60 * 1000);
    ```

    Export `async function getWeeklyStats(userId: string)`. Build:
    ```
    const { start } = lastNDaysRange(7);
    const todayStart = vietnamDayStart(new Date());
    const userObjId = new mongoose.Types.ObjectId(userId);

    const [agg] = await WorkoutLog.aggregate([
      { $match: { userId: userObjId, date: { $gte: start } } },
      { $group: { _id: null, exercises: { $sum: 1 }, kcal: { $sum: '$caloriesBurned' }, minutes: { $sum: '$durationMinutes' }, dayBuckets: { $addToSet: '$date' } } },
    ]);
    const [todayAgg] = await WorkoutLog.aggregate([
      { $match: { userId: userObjId, date: todayStart } },
      { $group: { _id: null, kcal: { $sum: '$caloriesBurned' } } },
    ]);

    return {
      days: agg ? agg.dayBuckets.length : 0,
      exercises: agg ? agg.exercises : 0,
      kcal: agg ? agg.kcal : 0,
      minutes: agg ? agg.minutes : 0,
      todayKcal: todayAgg ? todayAgg.kcal : 0,
      targetKcal: 300,
    };
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| mobile client → /api/workouts/* | Untrusted JSON body (durations, kcal, completedAt, possibly malicious userId) crosses here |
| /api/workouts/* → MongoDB | Per-user writes — userId MUST come from JWT, never body |
| Aggregation pipeline → User isolation | $match must filter by userId or cross-tenant leak occurs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07 | Elevation of Privilege | userId injection via request body | mitigate | Zod schema does NOT include `userId`; `.strict()` on schema rejects unknown keys (returns 400); controller extracts `userId` from `(req as AuthRequest).user.id`; service constructor takes userId as separate parameter (CRITICAL — this is the cross-tenant attack surface). Integration test case 3 verifies. |
| T-03-08 | Information Disclosure | Cross-tenant stats leak via aggregation | mitigate | `$match: { userId: ObjectId(userId) }` is FIRST stage of every pipeline; coerce to ObjectId via `new mongoose.Types.ObjectId(userId)` (string compare would not match the persisted ObjectId). Integration test case 6 verifies isolation. |
| T-03-09 | Tampering | NoSQL injection via exerciseId | mitigate | Zod regex `/^[0-9a-fA-F]{24}$/` on exerciseId before service touches DB |
| T-03-10 | Tampering | NoSQL injection via completedAt | mitigate | `z.string().datetime()` enforces ISO 8601; controller passes string to `new Date()` then to `vietnamDayStart` — both are scalar conversions, not query operators |
| T-03-11 | Denial of Service | Massive aggregation across full collection | mitigate | $match uses `{ userId, date: { $gte: 7-days-ago } }` — index `{ userId: 1, date: -1 }` (already exists per Phase 1 D-11) covers it; bounded result set |
| T-03-12 | Spoofing | Anonymous workout log creation | mitigate | `authenticate` middleware on POST route — 401 before handler |
| T-03-13 | Tampering | caloriesBurned/durationMinutes inflated to game leaderboards | accept | v1 has no leaderboards; max bounds (`max(600)` minutes, `max(10000)` kcal) in Zod limit blast radius for data integrity. v2 social features (ROADMAP §v2) would re-evaluate. |
| T-03-SC | Tampering | npm install of new dependencies | accept | No new dependencies added — `mongoose`, `zod`, `express`, `supertest`, `mongodb-memory-server` all from Phase 1/2 |
</threat_model>
