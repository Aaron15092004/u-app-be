## Spacing Scale

Declared values (must be multiples of 4). React Native uses logical pixels (dp), not px.

| Token | dp | Usage |
|-------|-----|-------|
| xs | 4 | Icon inner padding, inline element gaps |
| sm | 8 | Between label and value, icon gap from text |
| md | 16 | Default horizontal screen padding, card inner padding |
| lg | 24 | Section vertical gap, bottom sheet padding |
| xl | 32 | Major vertical rhythm between sections |
| 2xl | 48 | Above/below page headings |
| 3xl | 64 | Full-screen vertical centering offset |

Exceptions (this phase):

| Exception | Value | Reason |
|-----------|-------|--------|
| Touch target minimum | 44 dp | iOS HIG minimum — applies to pause/stop/complete buttons on timer screen and "Đánh dấu +1" habit buttons |
| Timer digit display | 80 dp font-size (not a spacing value) | Full-screen countdown readability on orange background |
| Habit row height | 72 dp | Comfortable tap target for HAB-03 check-in |
| BMI slider track height | 8 dp | Visual affordance without dominating layout — matches `sm` token |
| Weekly stats card gap | 12 dp | 4-column stat cards need tighter gutter than default md=16 |

## Typography

All font sizes are in logical pixels (dp), matching the established pattern in existing components.

| Role | Size (dp) | Weight | Line Height | Usage |
|------|-----------|--------|-------------|-------|
| Display | 28 | 700 (bold) | 1.2 | Screen headings: "Xuất sắc!", "Phân tích BMI", "Thói quen" section header |
| Heading | 20 | 700 (bold) | 1.2 | Exercise name on detail screen, stat section titles, BMI score value |
| Body | 16 | 400 (regular) | 1.5 | Exercise description text, habit tips, BMI advice text, input values |
| Label | 14 | 700 (bold) | 1.2 | Category filter chips, difficulty tags, stat labels ("Ngày tập", "Bài tập"), slider unit labels (cm / kg) |

Rules:
- Exactly 4 sizes, 2 weights (400 regular + 700 bold). No 500/600 weights.
- Timer countdown uses Display (28 dp) but rendered at 80 dp via StyleSheet (exception carved out in Spacing Exceptions above).
- Vietnamese text: no special font needed — system default renders Vietnamese diacritics correctly on iOS and Android.
- Source: consistent with existing components — ScreenHeader uses 24/700 for title (treat as Display), 16/400 for subtitle (Body). AuthInput uses 14/700 for label, 16/400 for input value.

## Screen Inventory and Interaction Contracts

### 3.1 Exercise List Screen (WO-01, WO-02, WO-03, WO-04, WO-11)

**Route:** `/(tabs)/exercises` (new tab)

**Primary visual anchor:** The category filter chip row — the horizontally-scrolling chips are the first interactive element the user sees and determine all content below them.

**Layout:**
- SafeAreaView > ScrollView (vertical)
- Screen heading: "Tập luyện" (Display/28/700)
- Weekly stats row: 4 stat cards horizontal — Ngày tập / Bài tập / kcal / Phút
- Daily challenge card: "Đốt 300 kcal hôm nay" + progress bar (accent fill)
- Category filter chips: Tất cả | Yoga | Cardio | Tạ | Giãn cơ — horizontal ScrollView, no wrapping
- Exercise FlatList: vertical, each card = thumbnail area + name + difficulty tag + duration + kcal

**Interaction:**
- Filter chip tap: active chip gets `bg-primary` background + white text. Inactive: `bg-surface` + border. No animation required — instant state swap.
- Exercise card tap: navigate to ExerciseDetail (`/(tabs)/exercises/[id]`)
- Scroll: FlatList with `keyExtractor={item => item._id}`, no pagination for v1 (100 items loads fine)

**States:**
- Loading: FlatList shows 3 skeleton card placeholders (gray shimmer rectangles at card dimensions)
- Empty per filter: show inline empty message (see Copywriting)
- Error: show error card with retry button (see Copywriting)

**Exercise card anatomy:**
- Thumbnail: 80x80 dp `bg-surface border border-gray-200 rounded-lg` — shows category icon (Ionicons) centered when imageUrl is null (D-42)
- Category icon map: Yoga → `body-outline`, Cardio → `bicycle-outline`, Tạ → `barbell-outline`, Giãn cơ → `fitness-outline`
- Exercise name: Body/16/400, 2-line max, ellipsis
- Difficulty tag: Label/14/700, colored pill — Dễ=#4CAF50, Trung bình=#FFA726, Khó=#EF5350
- Duration: Label/14/400, `text-text-secondary` — "15 phút"
- kcal: Label/14/400, `text-text-secondary` — "120 kcal"

### 3.2 Exercise Detail Screen (WO-05, WO-06)

**Route:** `/(tabs)/exercises/[id]`

**Primary visual anchor:** The sticky "Bắt đầu tập" button at the bottom — it is the terminal action that the entire screen leads toward.

**Layout:**
- SafeAreaView > ScrollView
- Back button (ScreenHeader with showBack=true)
- Thumbnail area: full-width 200dp tall, category icon centered on `bg-surface`
- Exercise name: Display/28/700
- Metadata row: difficulty tag + duration + kcal
- Section "Mô tả": Body/16/400/1.5 text block
- Section "Các động tác": ordered list of steps — step name (Body/16/700) + duration (Label/14/400 secondary)
- Section "Lưu ý": Body/16/400 italic, `text-text-secondary`
- Sticky bottom bar: "Bắt đầu tập" PrimaryButton (bg-primary, full-width, 52dp height, 16dp radius)

**Interaction:**
- "Bắt đầu tập" tap: navigate to ExerciseTimer (`/(tabs)/exercises/[id]/timer`), pass exercise data via route params

### 3.3 Exercise Timer Screen (WO-07, WO-08) — Orange Theme

**Route:** `/(tabs)/exercises/[id]/timer`

**Primary visual anchor:** The countdown digit display at center screen — the large 80dp MM:SS number is the dominant element the user watches for the entire session.

**Layout:**
- Full-screen, background `#FF6B35`
- StatusBar: light-content
- SafeAreaView top + bottom
- Top row: exercise name, centered, Body/16/700 white
- Center: countdown digits at 80dp/700/white — format `MM:SS`
- Below timer: progress ring (optional — thin circular track, white 20% opacity, fill white 80% opacity arc)
- Bottom controls (3 buttons, 44dp touch targets each):
  - Dừng (stop) — Ionicons `stop-circle-outline`, white, 32dp icon
  - Pause/Resume — Ionicons `pause-circle-outline` / `play-circle-outline`, white, 48dp icon (primary action)
  - (empty third slot, symmetric layout)

**Interaction:**
- Pause: freeze countdown. Icon switches to play. Resume from same value. (D-46)
- Stop ("Dừng"): trigger Alert.alert confirmation (see Copywriting). If confirmed → navigate back to ExerciseDetail, no WorkoutLog created. (D-47)
- Timer reaches 0: auto-navigate to ExerciseComplete without user action
- AppState 'background': auto-pause. On foreground: remain paused, show "Đã tạm dừng" subtitle below timer. (D-45)

**Zustand timer store shape** (for executor reference):
```
{ isRunning: boolean, isPaused: boolean, remainingSeconds: number, exerciseId: string | null }
```

### 3.4 Exercise Complete Screen (WO-09, WO-10) — Orange Theme

**Route:** `/(tabs)/exercises/[id]/complete`

**Primary visual anchor:** The trophy icon + "Xuất sắc!" heading pair — centered vertically on the orange background, this is the emotional reward moment.

**Layout:**
- Full-screen, background `#FF6B35`
- StatusBar: light-content
- Centered vertically:
  - Trophy icon: Ionicons `trophy-outline`, white, 64dp
  - Heading: "Xuất sắc!" — Display/28/700/white
  - Subtitle: "Bạn đã hoàn thành bài tập!" — Body/16/400/white
  - Stats summary: duration + kcal burned — Label/14/400/white, centered
- Bottom: "Hoàn tất" PrimaryButton — white background, `text-accent` label, full-width, 52dp height

**Interaction:**
- Screen mount: call `POST /api/workouts` to save WorkoutLog. Show ActivityIndicator overlay until API resolves. (WO-10)
- "Hoàn tất" tap: navigate back to Exercise List `/(tabs)/exercises`, reset timer Zustand store

### 3.5 Habit Tracking Screen (HAB-01–HAB-07)

**Route:** `/(tabs)/habits` (new tab)

**Primary visual anchor:** The habit list rows with "Đánh dấu +1" buttons — the 6 rows with their tap targets are the central daily interaction surface.

**Layout:**
- SafeAreaView > ScrollView
- Screen heading: "Thói quen" (Display/28/700)
- Progress summary: "X/6 hoàn thành — Y%" — Body/16/400 `text-text-secondary`
- Streak badge: Ionicons `flame-outline` accent-colored + "{N} ngày" Label/14/700 — top-right corner
- Habit list: 6 rows (no FlatList — static), each 72dp tall
- Weekly heatmap section heading: "Tuần này" (Heading/20/700)
- Heatmap: 7 cells T2 T3 T4 T5 T6 T7 CN — each cell 36x36dp, rounded-full, `bg-habit-active` or `bg-habit-inactive`
- Tips section: "Mẹo nhỏ" heading (Heading/20/700) + single tip card (Body/16/400)

**Habit row anatomy:**
- Left: habit icon (Ionicons, 24dp, `text-primary`) + habit name (Body/16/400)
- Right: "Đánh dấu +1" button (Label/14/700, `text-primary`, min 44dp touch target) — changes to checkmark icon `checkmark-circle` when completed (binary: done once today disables further taps — one check per day per habit via HabitLog unique index)
- Row separator: 1dp `bg-habit-inactive` line

**6 default habits (Vietnamese names, fixed):**
1. Uống 8 ly nước — icon: `water-outline`
2. Ăn 5 bữa rau củ — icon: `leaf-outline`
3. Tập luyện 30 phút — icon: `fitness-outline`
4. Ngủ đủ 8 tiếng — icon: `moon-outline`
5. Đọc sách 20 phút — icon: `book-outline`
6. Uống sữa hạt — icon: `cafe-outline`

**Interaction:**
- "Đánh dấu +1" tap: call `POST /api/habits/check-in`, optimistic update (mark row as done instantly), rollback on error
- Already-done row: greyed "Đánh dấu +1" disabled, checkmark icon visible
- Heatmap cells: non-interactive (display only)

**States:**
- Loading habit log: show skeleton rows (gray rectangles)
- Error: inline error text with retry

### 3.6 BMI Analysis Screen (BMI-01–BMI-06)

**Route:** `/(tabs)/bmi` or accessible from profile — implement as a standalone tab or modal, planner decides route

**Primary visual anchor:** The BMI result card with the color-segmented scale bar — the score value and position dot give users an immediate, scannable read of their health status.

**Layout:**
- SafeAreaView > ScrollView
- Screen heading: "Phân tích BMI" (Display/28/700)
- BMI result card (`bg-surface`, rounded-2xl, shadow):
  - BMI score: Heading/20/700 primary-colored
  - Category label: Body/16/400 — "Bình thường" etc.
  - BMI scale bar: horizontal, 4 color segments (bmi-underweight / bmi-normal / bmi-overweight / bmi-obese), 8dp track height, rounded ends, current position dot (white circle with primary border, 16dp)
  - Scale range labels: "15" … "40" at ends, Label/14/400 secondary
- Sliders section:
  - Heading: "Cập nhật số đo" (Heading/20/700)
  - Height slider: 100–220 cm, Ionicons `resize-outline`, current value Label/14/700 primary
  - Weight slider: 30–200 kg, Ionicons `barbell-outline`, current value Label/14/700 primary
  - "Lưu số đo" button: PrimaryButton filled, bg-primary, full-width (D-53)
- Health advice card: "Lời khuyên" heading + Body/16/400 advice text based on BMI category
- 30-day chart section:
  - Heading: "Lịch sử 30 ngày" (Heading/20/700)
  - Bar chart: use `react-native-gifted-charts` or `victory-native` — planner selects library. Bar color = bmi-normal (#4CAF50). Y-axis: BMI value. X-axis: date labels every 7 days.

**Interaction:**
- Slider drag: recalculate BMI instantly, update score display, update scale bar dot position, update advice text — all client-side, no API call (D-53)
- "Lưu số đo" tap: call `PATCH /api/bmi`, show loading state on button, on success show toast "Đã lưu!", update chart (D-53, D-54)
- Chart: read-only, no interaction

**States:**
- No BMI history (first use): slider at user's profile values (from AuthProvider user object), chart shows empty state message (see Copywriting)
- Loading chart: gray skeleton bar chart placeholder
- Save loading: "Lưu số đo" button shows ActivityIndicator

## Interaction States Matrix

Every interactive element must implement these states:

| State | Visual treatment |
|-------|-----------------|
| Default | As specified per component |
| Pressed | `opacity: 0.85` (consistent with PrimaryButton pattern) |
| Loading | ActivityIndicator replaces label/icon, element disabled |
| Disabled | `opacity: 0.5`, no press handler |
| Error | Red border or inline FormErrorText below element |
| Success | Toast ("Đã lưu!") auto-dismiss 2000ms, no screen change required |
