| Boundary | Description |
|---|---|
| Admin export -> printed bottles | Raw codes leave backend only in controlled export responses. |
| Mobile redeem -> backend | User-submitted codes must be normalized, hashed, rate-limited, and never logged. |
| Food scan -> AI cost boundary | Active entitlements raise daily quota but remain finite. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-02-01-01 | Information Disclosure | RedeemCode storage | mitigate | Tests must prove no persisted raw code fields are added. |
| T-02-01-02 | Tampering | Redemption | mitigate | Atomic compare-and-set on unused code plus unique `redeemCodeId` entitlement index. |
| T-02-01-03 | Denial of Service | AI scan quota | mitigate | 30/day entitlement quota and existing scan attempt counting; no unbounded bypass. |
| T-02-01-04 | Elevation of Privilege | Admin campaign APIs | mitigate | Mount admin endpoints only under `authenticate, requireAdmin`. |
</threat_model>
