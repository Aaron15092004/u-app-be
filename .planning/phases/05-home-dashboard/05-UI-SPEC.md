# Phase 5 — UI Design Contract
## Home Dashboard, Profile & Notifications

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (React Native StyleSheet.create + NativeWind v4 classes) |
| Preset | not applicable |
| Component library | none — custom components only |
| Icon library | @expo/vector-icons (Ionicons) |
| Font | System default (no custom font) |

**Note:** This is a React Native / Expo SDK 54 app. No shadcn/radix — all components are hand-rolled using `StyleSheet.create()` following the patterns established in Phases 2–4. The token source of truth is `mobile/src/constants/colors.ts`.

---

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline divider padding, badge inner gap |
| sm | 8px | Compact element spacing, between stat columns, tag chips |
| md | 16px | Default card padding, screen horizontal padding, section gaps |
| lg | 24px | Screen top padding, section heading margin, modal padding |
| xl | 32px | Between major sections (dashboard widget rows) |
| 2xl | 48px | Bottom safe area padding, scroll end padding |
| 3xl | 64px | Not used in Phase 5 |

Exceptions:
- Touch targets (water +/− buttons, quick action buttons, notification toggles): minimum 44px height — accessibility requirement.
- Tab bar icon area: 28px icon size (matches existing 4 tabs).
- Progress bar track height: 8px (matches DailyChallengeCard pattern).
- Card border radius: 12px (standard) or 16px (primary cards — matches WeeklyStatCard / DailyChallengeCard).

---

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 | 1.5 | Secondary labels, habit names, menu item subtitles, notification body copy |
| Label | 16px | 400 | 1.5 | Menu item titles, form field values, card body text |
| Heading | 20px | 700 | 1.2 | Card stat values (kcal, water count), section headings |
| Display | 24px | 700 | 1.2 | Screen greeting, profile name, BMI widget number |

Additional confirmed sizes from existing components (do not deviate):
- Unit suffix on stat cards: 12px weight 400 color TEXT_SECONDARY (matches WeeklyStatCard.unit)
- Progress bar label: 12px weight 400 right-aligned color TEXT_SECONDARY (matches DailyChallengeCard.kcalLabel)
- Badge milestone label: 12px weight 600 (below badge icon in Profile)
- Button label: 16px weight 700 (matches PrimaryButton)
- Screen section subtitle: 16px weight 400 color TEXT_SECONDARY (matches ScreenHeader.subtitle)

---

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #F5F5F5 | Screen backgrounds, ScrollView backgrounds |
| Secondary (30%) | #FFFFFF | All cards, water log items, profile menu rows, form input backgrounds |
| Accent (10%) | #4CAF50 | Reserved list below |
| Destructive | #EF5350 | Logout confirmation button (destructive Alert style only) |

**Accent (#4CAF50) reserved for:**
1. Active tab bar icon tint
2. Progress bar fill (kcal goal, water glasses, macro nutrition bars)
3. Achievement badge background when unlocked: #E8F5E9 (bg) + #4CAF50 (icon/text)
4. Quick action button fill (3 buttons: Quét bữa ăn / Bắt đầu tập / Thói quen) — filled PrimaryButton variant
5. Toggle switch track when ON (notification settings)
6. "Bật thông báo" CTA on rationale screen

**Secondary semantic colors (supporting, not accent):**
- #FF6B35 (STREAK_BADGE / ACCENT): streak flame icon in Profile stats row only. Not used for buttons or interactive elements in Phase 5.
- #BDBDBD: Locked achievement badge icon and background (#F5F5F5 bg + #BDBDBD icon). Tertiary disabled text.
- #E0E0E0: Progress track (empty fill background). Matches existing DailyChallengeCard and HabitRow patterns.
- #64B5F6: Macro "Chất béo" color indicator (matches NutritionSummaryCard pattern).
- #FFA726: Macro "Carbs" color indicator (matches NutritionSummaryCard pattern).

**Shop banner:** Green gradient banner — linear gradient from #4CAF50 to #388E3C (PRIMARY → PRIMARY_DARK). White text and arrow icon on top.

## Screen Inventory & Interaction Contracts

### Screen 1: Home Dashboard (`app/(tabs)/index.tsx`)

**Layout:** ScrollView (bounces), `paddingHorizontal: 16`, `paddingTop: 16`, `paddingBottom: 48`.

**Section order (top to bottom):**
1. Greeting row + notification bell
2. Today Summary cards row
3. Ủ Shop banner
4. Quick Actions row
5. BMI widget card
6. Nutrition Summary (macro progress bars)

**1.1 Greeting Row**
- Left: "Xin chào, [user.name]!" — Display 24px weight 700 color TEXT (#212121)
- Left sub-line (optional date): "Thứ Hai, 19 tháng 5" — Body 14px weight 400 TEXT_SECONDARY
- Right: Bell icon button (`notifications-outline` Ionicons, size 24, color TEXT). Tap = no-op in Phase 5 (badge count not in scope). Touch target: 44×44px.

**1.2 Today Summary Cards Row**
- 3 equal-width cards in a row (`flex: 1` each, `gap: 8`).
- Reuse `WeeklyStatCard` component.
- Card 1: value = kcalConsumed, unit = "kcal", label = "Đã ăn"
- Card 2: value = waterGlasses / waterGoal, unit = "ly", label = "Nước". **Tappable** — `TouchableOpacity`, `onPress` → push `/(home)/water`. Active opacity 0.8.
- Card 3: value = workoutMinutes, unit = "phút", label = "Tập luyện"
- Card background: #FFFFFF, borderRadius 16, paddingVertical 12, paddingHorizontal 8. No elevation/shadow needed here (flat row).

**1.3 Ủ Shop Banner**
- Full-width card, borderRadius 12, `marginVertical: 16`.
- Background: linear gradient `#4CAF50 → #388E3C` (left to right) via `expo-linear-gradient`.
  - Fallback if expo-linear-gradient not installed: solid `#4CAF50` background.
- Left content: Icon `storefront-outline` size 28 white + text block: heading "Ủ Shop" 16px weight 700 white + subtext "Khám phá sản phẩm sức khỏe" 12px weight 400 rgba(255,255,255,0.85).
- Right: `chevron-forward` Ionicons size 20 white.
- `padding: 16`. Tap → `Linking.openURL(shopUrl)`. Feedback: `TouchableOpacity` activeOpacity 0.85.
- Loading state: shopUrl fetched via TanStack Query (cache 1 hour). While loading: banner shows with placeholder text, chevron hidden. On error: banner hidden (do not show broken banner).

**1.4 Quick Actions Row**
- 3 equal-width `Pressable` cards in a row, `gap: 8`.
- Each card: white background, borderRadius 12, padding 16, alignItems center, elevation 2 (Android) / shadow (iOS: shadowColor #000, shadowOffset {0,1}, shadowOpacity 0.08, shadowRadius 4).
- Content: Ionicons icon (size 28, color PRIMARY #4CAF50) + label below (14px weight 600 color TEXT).
- Card 1: icon `scan-outline`, label "Quét bữa ăn" → `router.push('/(food)/scan')`
- Card 2: icon `barbell-outline`, label "Bắt đầu tập" → `router.push('/(tabs)/exercises')`
- Card 3: icon `checkmark-circle-outline`, label "Thói quen" → `router.push('/(tabs)/habits')`
- Press feedback: `opacity: 0.85` on pressed state.

**1.5 BMI Widget Card**
- White card, borderRadius 12, padding 16, elevation 2 / shadow (same as Quick Actions).
- Section heading "Chỉ số BMI" 14px weight 700 TEXT, `marginBottom: 12`.
- Large BMI number: 28px weight 700 color based on category (BMI_UNDERWEIGHT / BMI_NORMAL / BMI_OVERWEIGHT / BMI_OBESE from colors.ts).
- Category label: 14px weight 600 same category color. e.g. "Bình thường".
- Tap → `router.push('/(tabs)/bmi')`. Entire card is `Pressable`, activeOpacity 0.95.
- Empty state: if no BMIRecord exists, show "Chưa có dữ liệu BMI" 14px TEXT_SECONDARY + "Cập nhật ngay →" 14px weight 600 PRIMARY.

**1.6 Nutrition Summary Card**
- White card, borderRadius 12, padding 16, elevation 2 / shadow.
- Section heading "Dinh dưỡng hôm nay" 14px weight 700 TEXT, `marginBottom: 12`.
- 4 progress bars stacked vertically, `gap: 12`.
- Each row: label left (14px weight 400 TEXT) + value+unit right (12px weight 400 TEXT_SECONDARY) + bar below (height 8px, borderRadius 4, track #E0E0E0, fill color per macro).
  - Calo: fill #4CAF50. Goal = 2000 kcal default (phase: no user kcal goal config, use constant).
  - Protein: fill #4CAF50. Goal = 50g.
  - Carbs: fill #FFA726. Goal = 250g.
  - Chất béo: fill #64B5F6. Goal = 70g.
- Progress fill width = `Math.min(1, actual / goal) * 100%`.
- Empty state (no food logs today): all bars at 0%. Show "Chưa có bữa ăn nào hôm nay" body text below bars.

### Screen 2: Water Log (`app/(home)/water.tsx`)

**Navigation:** Stack push from Home (water summary card tap). Uses `ScreenHeader` with `showBack: true`, title "Nhật ký nước".

**Layout:** ScrollView, `paddingHorizontal: 16`, `paddingTop: 16`.

**Top section — daily goal card:**
- White card, borderRadius 12, padding 16, elevation 2.
- Large display: "[X] / [goal]" — X in 36px weight 700 PRIMARY, goal in 20px weight 400 TEXT_SECONDARY.
- Unit label "ly hôm nay" 14px weight 400 TEXT_SECONDARY below.
- Progress bar: full-width, height 12px, borderRadius 6, track #E0E0E0, fill #4CAF50. Fill = min(X/goal, 1) × 100%.
- Goal reached state: fill #4CAF50, label changes to "Đã đạt mục tiêu! Tuyệt vời" 14px weight 600 PRIMARY.

**Middle section — controls:**
- Row centered with `gap: 24`. Three elements inline:
  - Minus button: circular 52×52 Pressable, border 1.5px PRIMARY, icon `remove` Ionicons size 24 PRIMARY. Disabled when X === 0 (opacity 0.4, not pressable).
  - Water count display: 48px weight 700 TEXT — the current count repeated (redundant but good UX anchor).
  - Plus button: circular 52×52 filled PRIMARY, icon `add` Ionicons size 24 white.
- Both buttons: `activeOpacity: 0.8`. Loading state on mutation: show `ActivityIndicator` inside button, disable both buttons.

**Bottom section — today's log list:**
- Section heading "Lịch sử hôm nay" 14px weight 700 TEXT, `marginTop: 24`, `marginBottom: 8`.
- FlatList of logged glasses (reverse chronological by `loggedAt`).
- Each item: white row, borderRadius 8, padding 12, marginBottom 8, `flexDirection: row`.
  - Left: `water-outline` Ionicons size 20 #64B5F6.
  - Right: "Ly [N]" 14px weight 600 TEXT + time string (HH:MM) 12px TEXT_SECONDARY.
- Empty state: "Chưa uống ly nào hôm nay. Hãy bắt đầu nhé!" 14px TEXT_SECONDARY centered, `marginTop: 24`.

**Tab bar entry:** 5th tab. Icon `person` (focused) / `person-outline` (inactive). Label "Hồ sơ". Tab bar tint: PRIMARY #4CAF50 active, TEXT_SECONDARY #757575 inactive. (Source: CONTEXT.md D-76)

**Section order:**
1. Avatar + name + email block
2. Stats row (streak / workouts / kcal burned)
3. Achievement badges row
4. Menu items list
5. Logout button

**3.1 Avatar + Name + Email**
- Centered column layout.
- Avatar: 80×80 circular placeholder. If no avatar URL: filled circle background PRIMARY (#4CAF50) with user initials 28px weight 700 white. If avatar URL: `Image` component with borderRadius 40.
- Name: 20px weight 700 TEXT, `marginTop: 12`.
- Email: 14px weight 400 TEXT_SECONDARY, `marginTop: 4`.

**3.2 Stats Row**
- 3 equal-width columns in a row, `marginTop: 24`, white card borderRadius 12 padding 16 elevation 2.
- Each column: value (20px weight 700 TEXT) + label (12px weight 400 TEXT_SECONDARY below).
  - Col 1: streakDays value + "ngày streak". Flame icon `flame-outline` size 16 STREAK_BADGE (#FF6B35) inline before value.
  - Col 2: totalWorkouts value + "bài tập".
  - Col 3: totalKcal value + "kcal đốt".
- Dividers: 1px #E0E0E0 vertical lines between columns.

**3.3 Achievement Badges Row**
- White card, borderRadius 12, padding 16, elevation 2, `marginTop: 16`.
- Section heading "Thành tích" 14px weight 700 TEXT, `marginBottom: 12`.
- 4 badges in a row, equal spacing (`justifyContent: 'space-around'`).
- Each badge (column layout):
  - Circle container 56×56, borderRadius 28.
  - Unlocked (streak >= milestone): background #E8F5E9, icon `medal-outline` Ionicons size 28 color #4CAF50.
  - Locked (streak < milestone): background #F5F5F5, icon `medal-outline` Ionicons size 28 color #BDBDBD.
  - Label below circle: milestone label (12px weight 600). Unlocked: #4CAF50. Locked: #BDBDBD.

**3.4 Menu Items**
- White card, borderRadius 12, `marginTop: 16`, no padding on container.
- Each menu row: `height: 52`, `paddingHorizontal: 16`, `flexDirection: row`, `alignItems: center`.
- Left: Ionicons icon 20px TEXT_SECONDARY + label 16px weight 400 TEXT, `gap: 12`.
- Right: `chevron-forward` Ionicons 16px #BDBDBD.
- Separator between rows: 1px #E0E0E0 line (do NOT add after last row).
- Rows:
  1. `create-outline` "Chỉnh sửa hồ sơ" → `router.push('/(tabs)/profile/edit')`
  2. `notifications-outline` "Cài đặt thông báo" → `router.push('/(tabs)/profile/notifications')`
  3. `help-circle-outline` "Trợ giúp & Hỗ trợ" → `router.push('/(tabs)/profile/help')`
- Press feedback: row background changes to #F5F5F5 on pressed state.

**3.5 Logout Button**
- `PrimaryButton` variant "outlined", label "Đăng xuất", `marginTop: 24`, full width.
- On tap: `Alert.alert('Đăng xuất', 'Bạn có chắc muốn đăng xuất không?', [{text: 'Hủy', style: 'cancel'}, {text: 'Đăng xuất', style: 'destructive', onPress: auth.logout}])`

### Screen 4: Edit Profile (`app/(tabs)/profile/edit.tsx`)

**Navigation:** Push from Profile menu. `ScreenHeader` with `showBack: true`, title "Chỉnh sửa hồ sơ".

**Layout:** `KeyboardAvoidingView` wrapping `ScrollView`, `paddingHorizontal: 16`, `paddingTop: 16`.

**Form fields (white card, borderRadius 12, padding 16, elevation 2):**
Each field uses a consistent field row pattern:
- Label: 14px weight 600 TEXT, `marginBottom: 6`.
- Input: height 48, borderRadius 8, borderWidth 1, border color #E0E0E0 (inactive) / PRIMARY (focused), background #FFFFFF, paddingHorizontal 12, fontSize 16 weight 400 TEXT.

Fields (top to bottom):
1. **Tên hiển thị** — TextInput, placeholder "Nhập tên của bạn", returnKeyType "next".
2. **Chiều cao (cm)** — TextInput, keyboardType "numeric", placeholder "170", returnKeyType "next".
3. **Cân nặng (kg)** — TextInput, keyboardType "numeric", placeholder "65", returnKeyType "next".
4. **Mục tiêu sức khỏe** — 3-option selector (reuse GoalCard pattern): "Giảm cân" / "Duy trì" / "Tăng cân". Row of 3 cards, each 80px tall, borderRadius 12, border 2px. Selected: border PRIMARY, background rgba(76,175,80,0.1). Unselected: border #E0E0E0, background #FFFFFF.
5. **Mục tiêu nước mỗi ngày (ly)** — Row with minus/plus stepper (same circular button pattern as Water Log controls), current value displayed center 20px weight 700 TEXT. Range: 4–16 glasses.

**Save button:** `PrimaryButton` variant "filled", label "Lưu thay đổi", `marginTop: 24`, full width. Shows `ActivityIndicator` while PATCH /api/users/profile in-flight.

**Success feedback:** Toast-style notification "Đã lưu hồ sơ" OR simply `router.back()` on success. Use `Alert.alert('Thành công', 'Hồ sơ đã được cập nhật.')` for simplicity (matches existing auth screens pattern).

### Screen 5: Notification Settings (`app/(tabs)/profile/notifications.tsx`)

**Navigation:** Push from Profile menu. `ScreenHeader` with `showBack: true`, title "Cài đặt thông báo".

**Layout:** ScrollView, `paddingHorizontal: 16`, `paddingTop: 16`.

**White card, borderRadius 12, padding 0, elevation 2** containing all settings rows.

**Row pattern:**
- Each row: `height: 64`, `paddingHorizontal: 16`, `flexDirection: row`, `alignItems: center`, `justifyContent: 'space-between'`.
- Left block: label 16px weight 600 TEXT + sublabel 12px weight 400 TEXT_SECONDARY below.
- Right: Switch component (React Native core). `trackColor={{ false: '#E0E0E0', true: '#4CAF50' }}`, `thumbColor: '#FFFFFF'`.
- Separator: 1px #E0E0E0 between rows (not after last row).

**Rows:**
1. Toggle row — Label "Nhắc uống nước", sublabel "Nhắc bạn uống nước mỗi ngày". Switch for `waterReminder`.
2. Time row (conditional — only visible when `waterReminder === true`): Label "Giờ nhắc uống nước", sublabel = selected time string e.g. "08:00". Right side: time display "08:00" 16px weight 700 PRIMARY + `chevron-forward` 16px #BDBDBD. Tap → open time picker (DateTimePicker, mode 'time').
3. Separator.
4. Toggle row — Label "Nhắc tập luyện", sublabel "Nhắc bạn bắt đầu tập mỗi ngày". Switch for `workoutReminder`.
5. Time row (conditional — only visible when `workoutReminder === true`): Label "Giờ nhắc tập luyện", sublabel = selected time string e.g. "07:00". Same tap pattern.

**Time picker behavior:** Use `@react-native-community/datetimepicker` (already installed as Expo dependency). Show inline on iOS, modal on Android. On confirm: update local state → debounce 800ms → PATCH /api/users/notifications.

**Save is automatic** (no explicit Save button). Each toggle change and time picker confirm triggers PATCH immediately (with debounce on time).

**Info text at bottom:** "Streak alert sẽ tự động gửi lúc 20:00 mỗi ngày nếu bạn chưa hoàn thành thói quen." 12px TEXT_SECONDARY, `marginTop: 16`, centered.

## Interaction States

### Loading States
- Home dashboard: on initial load, show skeleton placeholders — gray `#E0E0E0` animated opacity 0.4→1 placeholder blocks matching card dimensions. Use `Animated.loop(Animated.sequence(...))`. Duration: 800ms per cycle.
- Water +/- buttons: `ActivityIndicator` inside button, both buttons disabled during mutation.
- Edit Profile save: `PrimaryButton` `loading={true}` state (existing component supports this).
- Profile stats: if no data yet, show `—` as value placeholder.

### Error States
- All `useQuery` hooks: on error → show inline error card below relevant section. Card: white, borderRadius 12, padding 16, row with `warning-outline` Ionicons size 20 #FFA726 + "Có lỗi xảy ra. Thử lại" 14px TEXT + retry `TouchableOpacity` "Thử lại" 14px weight 600 PRIMARY right.
- PATCH mutations (edit profile, notification settings): on error → `Alert.alert('Lỗi', 'Có lỗi xảy ra. Vui lòng thử lại.')`.

### Empty States
