> Visual and interaction contract for Phase 4 frontend screens.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|

| Preset | not applicable (React Native, not web) |

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tight inline padding |
| sm | 8px | Tag pill gap, icon-to-text gap |
| md | 16px | Default horizontal screen padding, list item vertical padding |
| lg | 24px | Section vertical gap, button height padding |
| xl | 32px | Major section gap (between nutrition cards) |
| 2xl | 48px | Screen top padding on content screens |
| 3xl | 64px | Camera bottom control area bottom padding |

Exceptions:
- Camera capture button diameter: 72px (touch target requirement for primary action)
- Camera gallery/flash buttons: 52px diameter (minimum 44px touch target, rounded)
- Scan frame corner brackets: 40px arm length, 4px stroke width
- Tab bar height: system default (~83px on iPhone with home indicator)

## Color

All values from `mobile/src/constants/colors.ts` (established Phase 3).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #FFFFFF | Screen backgrounds on result/search/diary screens |
| Secondary (30%) | #F5F5F5 (`BACKGROUND`) | Section dividers, detail row background, search screen list background |
| Accent (10%) | #4CAF50 (`PRIMARY`) | Reserved list below |
| Destructive | #EF5350 | "Chụp lại" outlined button border only (retry = mild destructive intent) |

**Accent (#4CAF50) reserved for:**
1. Capture button on camera screen (large circle background)
2. "Xác nhận & Lưu" filled CTA button background
3. Tag pills on AI result screen (background + text)
4. Kcal summary card background on AI result screen
5. Search bar focus border / active state
6. Food diary date header accent line
7. Tab bar icon for "Bữa ăn" tab (active state, follows global tab pattern)

**Camera screen is the only dark-theme screen:**
- Camera screen background: #000000 (full black — `bg-black`)
- Camera screen surface (header area): #1C2033 (dark navy, ~20% opacity overlay)
- Camera screen text: #FFFFFF
- Camera screen back icon: #FFFFFF
- Camera gallery/flash button background: rgba(255,255,255,0.15) (frosted pill)
- Scan frame corner brackets: #4CAF50 (green — confirmed in Figma iPhone 16 - 8.png)

**Micro-nutrient row colors (AI result detail section):**
- Chất xơ: #4CAF50 (green bar/label) — fiber is positive
- Đường: #FFA726 (orange warning) — sugar monitoring
- Natri: #EF5350 (red caution) — sodium monitoring
- Vitamin C: #64B5F6 (blue positive) — reuses BMI_UNDERWEIGHT blue for vitamin/positive micro

**Source:** Figma mockup (iPhone 16 - 7.png, iPhone 16 - 8.png) + existing `colors.ts` tokens.

## Screen Contracts

### Screen 1: Camera Scan (FOOD-01/02/03)

**Route:** `app/(food)/scan.tsx`
**Theme:** Dark (full screen, camera preview behind all UI)
**Entry point:** "Quét bữa ăn" button on `(tabs)/index.tsx` (D-69)

**Layout:**

```
┌─────────────────────────────┐
│  ← Quét bữa ăn              │  ← ScreenHeader variant (white text, dark bg)
│  Chụp ảnh để phân tích...   │
│                             │
│  ┌─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┐  │
│  │    [SCAN FRAME]         │  ← Green corner brackets only, no fill
│  │                         │
│  │   camera-outline icon   │
│  │ Căn chỉnh bữa ăn vào    │
│  │ khung                   │
│  └─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┘  │
│                             │
│  Nhấn nút chụp hoặc chọn    │
│  từ thư viện                │
│  [gallery]  [CAPTURE]  [⚡] │  ← Bottom control row
└─────────────────────────────┘
```

**Scan frame dimensions:** 260px wide × 260px tall, centered, green corner brackets (#4CAF50), arm length 40px, stroke 4px, border-radius 8px at corners.

**Scan frame hint text:** Ionicons `camera-outline` at 40px + text "Căn chỉnh bữa ăn vào khung" (14px, #FFFFFF, weight 400, center aligned).

**Bottom control row (fixed bottom, 24px from bottom safe area):**
- Gallery button: 52px circle, rgba(255,255,255,0.15) background, Ionicons `image-outline` 24px white
- Capture button: 72px circle, #4CAF50 background, Ionicons `radio-button-on` or `ellipse` 28px white, 3px white ring border (4px gap outside circle)
- Flash button: 52px circle, rgba(255,255,255,0.15) background, Ionicons `flash-outline` / `flash` 24px white (toggle state)

**Interactions:**
- Tap capture → compress with `expo-image-manipulator` (max 800×800, JPEG 0.7) → POST `/api/food/scan` → navigate to `result.tsx` with response data
- Tap gallery → `expo-image-picker` launchImageLibraryAsync → same compress + POST flow
- Tap flash → toggle flash mode (Ionicons `flash-outline` ↔ `flash`, button bg brightens to rgba(255,255,255,0.25))
- Loading state: disable all buttons, show `ActivityIndicator` white centered on capture button area
- Rate limit 429 response: show Alert "Bạn đã quét 20 lần hôm nay. Vui lòng thử lại vào ngày mai."

**Camera permission:** If not granted, show permission request card (centered, white text on dark bg): "Ứng dụng cần quyền truy cập camera để quét bữa ăn" + `PrimaryButton` "Cấp quyền" + link "Chọn từ thư viện" below.

### Screen 2: AI Result (FOOD-04/05/06)

**Route:** `app/(food)/result.tsx`
**Theme:** Light (white background)
**Entry:** Pushed from camera scan after successful AI response

**Layout top to bottom:**

1. **Header** — `ScreenHeader` reuse: title "Kết quả phân tích", subtitle "Thông tin dinh dưỡng bữa ăn", showBack=true
2. **Food image** — if image URI available from picker/camera: `Image` component, width 100%, height 200px, border-radius 0 (full bleed)
3. **Food name + tags row** — food name (24px/700/#212121) + green leaf Ionicons `leaf-outline` 16px #4CAF50 inline. Tags below: horizontal ScrollView of pill badges (background: #E8F5E9, text: #4CAF50, 14px/400, paddingH 8px, paddingV 4px, borderRadius 12px)
4. **Tổng quan dinh dưỡng section** — Section heading "Tổng quan dinh dưỡng" (16px/700/#212121). Green kcal card: `backgroundColor: '#4CAF50'`, borderRadius 16px, padding 16px, "Tổng calo" label (14px/400/white), kcal value (36px/700/white), progress sub-label "X% mục tiêu hàng ngày" (14px/400/rgba(255,255,255,0.8)). Below: 3-column macro row (Protein / Carbs / Chất béo), each: icon (Ionicons) + value (20px/700/#212121) + label (12px/400/#757575)
5. **Thông tin chi tiết section** — Section heading "Thông tin chi tiết" (16px/700/#212121). 4 rows (Chất xơ / Đường / Natri / Vitamin C): label (16px/400/#212121) left, value+unit (16px/400/#757575) right, 1px #F0F0F0 divider between rows
6. **CTA buttons** (fixed bottom, paddingH 16px, paddingBottom 24px + safe area):
   - `PrimaryButton` variant="filled" label="Xác nhận & Lưu" (full width)
   - `PrimaryButton` variant="outlined" label="Chụp lại" (full width, 8px gap above)

**Interactions:**
- "Xác nhận & Lưu" → POST `/api/food/logs` → show success toast "Đã lưu bữa ăn!" → router.push `/food/diary`
- "Chụp lại" → router.back() (returns to camera screen, discards result — no confirmation needed per FOOD-06)
- Loading on "Xác nhận & Lưu": button shows `ActivityIndicator`, disable both buttons

**Macro icons:**
- Protein: Ionicons `fitness-outline` (or `barbell-outline`) — color #4CAF50
- Carbs: Ionicons `leaf-outline` — color #FFA726
- Chất béo: Ionicons `water-outline` — color #64B5F6

### Screen 3: Manual Food Search (FOOD-07/08)

**Route:** `app/(food)/search.tsx`
**Theme:** Light (white background, #F5F5F5 list area)
**Entry:** "Tìm kiếm thủ công" link from camera screen OR from diary screen

**Layout top to bottom:**

1. **Header** — `ScreenHeader` title "Tìm kiếm thực phẩm", subtitle "Tìm trong cơ sở dữ liệu Việt Nam", showBack=true
2. **Search bar** — `TextInput` styled: height 48px, borderRadius 12px, border 1.5px solid #E0E0E0, paddingH 12px, Ionicons `search-outline` 20px #757575 left icon, placeholder "Tìm theo tên món ăn..." (#BDBDBD), focus border #4CAF50. Background: white.
3. **Results FlatList** — background #F5F5F5, each item: white card, borderRadius 12px, marginH 16px, marginBottom 8px, paddingH 16px, paddingV 12px. Row: food name (16px/600/#212121) left, kcal value (14px/400/#757575) right "Xkcal/100g". Tap → bottom sheet serving size input.
4. **Empty state** (when query has text but no results): centered, Ionicons `search-outline` 48px #BDBDBD, heading "Không tìm thấy kết quả" (16px/600/#212121), body "Thử từ khoá khác hoặc quét ảnh bữa ăn" (14px/400/#757575)
5. **Initial state** (no query yet): centered hint, Ionicons `restaurant-outline` 48px #BDBDBD, "Nhập tên món ăn để tìm kiếm" (14px/400/#757575)

**Serving size bottom sheet** (opens on item tap):
- Background: white, top borderRadius 20px, handle bar 40px×4px #E0E0E0 centered
- Food name heading (20px/700/#212121)
- Kcal per 100g sub-label (14px/400/#757575)
- `TextInput` for serving size: "Khối lượng (g)" label above, value default 100, keyboardType="numeric", borderRadius 12px
- Live calc row: "Dinh dưỡng ước tính:" label + calculated kcal (20px/600/#4CAF50)
- `PrimaryButton` variant="filled" label="Thêm vào nhật ký" full width, marginBottom 24px + safe area

**Interactions:**
- Input debounced 300ms → GET `/api/food/items?q={query}` (min 2 chars to trigger)
- Tap food item → open bottom sheet with serving size input
- "Thêm vào nhật ký" → POST `/api/food/logs` with aiProvider='manual' → success toast "Đã lưu bữa ăn!" → close sheet

### Screen 4: Food Diary (FOOD-09)

**Route:** `app/(food)/diary.tsx`
**Theme:** Light (white background)
**Entry:** "Nhật ký ăn" button on `(tabs)/index.tsx` (D-69)

**Layout top to bottom:**

1. **Header** — `ScreenHeader` title "Nhật ký bữa ăn", subtitle "Lịch sử các bữa ăn của bạn", showBack=true
2. **Date selector row** — horizontal scroll of date pills (today + 6 days back). Each pill: 44px height, borderRadius 22px, active: #4CAF50 background + white text (14px/600), inactive: #F5F5F5 background + #212121 text (14px/400). Format: "Hôm nay", "T7 17/5", "T6 16/5" etc.
3. **Daily kcal summary card** — `backgroundColor: '#F5F5F5'`, borderRadius 16px, marginH 16px, padding 16px. "Tổng calo hôm nay" label (14px/400/#757575), kcal value (28px/700/#212121), goal sub-text "/ 2,000 kcal mục tiêu" (14px/400/#757575).
4. **Meal list FlatList** — each item: white background, marginH 16px, marginBottom 8px, borderRadius 12px, padding 16px. Row layout: food name (16px/600/#212121), timestamp (12px/400/#BDBDBD), kcal right (16px/600/#4CAF50 + "kcal" in 12px/400/#757575). Divider 1px #F0F0F0 between items within same day.
5. **Empty state** (no meals logged for selected date): centered, Ionicons `restaurant-outline` 56px #BDBDBD, heading "Chưa có bữa ăn nào" (18px/600/#212121), body "Quét bữa ăn hoặc tìm kiếm thủ công để ghi lại" (14px/400/#757575), `PrimaryButton` label="Quét bữa ăn" below (navigates to scan screen).

**Interactions:**
- Tap date pill → GET `/api/food/logs?date=YYYY-MM-DD` → reload list
- Swipe left on meal item → reveal "Xóa" destructive action (red #EF5350 background, white text "Xóa", Ionicons `trash-outline`)
- Tap "Xóa" → Alert confirmation: title "Xóa bữa ăn", message "Bữa ăn này sẽ bị xóa vĩnh viễn. Bạn có chắc không?", buttons [Hủy / Xóa (destructive)]

## Navigation Contract

**Route group:** `app/(food)/` — modal/stack, NOT a tab

**Routes:**
- `app/(food)/_layout.tsx` — Stack layout, `headerShown: false`
- `app/(food)/scan.tsx` — Camera scan screen (FOOD-01/02/03)
- `app/(food)/result.tsx` — AI result screen (FOOD-04/05/06), receives nutrition data via router params or Zustand store
- `app/(food)/search.tsx` — Manual search (FOOD-07/08)
- `app/(food)/diary.tsx` — Food diary (FOOD-09)

**Zustand store:** `stores/foodScanStore.ts`
- State: `scanResult: NutritionResult | null`, `isScanning: boolean`, `pendingImageUri: string | null`
- Camera screen sets `pendingImageUri` and `isScanning`, result screen reads `scanResult`
- Store clears on "Chụp lại" (back action)

**Tab bar entry (temporary, D-69):** Add to `(tabs)/index.tsx`:
- "Quét bữa ăn" button → `router.push('/(food)/scan')`
- "Nhật ký ăn" button → `router.push('/(food)/diary')`
- These buttons are removed in Phase 5 when home dashboard is built

## Interaction States

### Camera scan loading
- Capture button: `ActivityIndicator` white (size=large) replaces circle content
- All three bottom buttons: `opacity: 0.4`, `pointerEvents: 'none'`
- Overlay text: "Đang phân tích..." centered below scan frame, #FFFFFF, 14px/400

### AI result save loading
- "Xác nhận & Lưu" button: `ActivityIndicator` green
- "Chụp lại" button: `opacity: 0.4`, disabled

### Search loading (debounced GET)
- `ActivityIndicator` #4CAF50 centered below search bar, replacing list content

### Delete swipe-to-delete
- Swipe gesture: react-native Animated API (no extra lib needed)
- Reveal: red (#EF5350) background, "Xóa" (white, 14px/600), Ionicons `trash-outline` 20px white
- Confirm via Alert (do not auto-delete on swipe)
