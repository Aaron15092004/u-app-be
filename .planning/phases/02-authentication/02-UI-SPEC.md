# Phase 2 — Authentication: UI Design Contract

> Visual and interaction contract for Phase 2 (Onboarding + Auth + Profile Setup).
> Generated by gsd-ui-researcher. Source of truth for gsd-planner and gsd-executor.

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | none (no shadcn — React Native) | platform constraint |
| Component library | NativeWind v4 (Tailwind-for-RN) | D-18, tailwind.config.js |
| Icon library | `@expo/vector-icons` (Ionicons subset) | standard Expo ecosystem |
| Font | System default sans-serif (SF Pro iOS / Roboto Android) | Project.md: "clean sans-serif" |
| Preset | not applicable | N/A |

**Note:** NativeWind v4 classes map 1:1 to Tailwind tokens. All class names in this spec use Tailwind syntax. React Native does not support web-only properties (e.g. `box-shadow` → use `shadow-*` RN classes).

---

## Spacing Scale

All values are multiples of 4. NativeWind uses numeric notation (`p-4` = 16px, `p-6` = 24px, etc.).

| Token | Value | Tailwind Class | Usage in Auth Phase |
|-------|-------|----------------|---------------------|
| xs | 4px | `p-1` / `gap-1` | Icon-to-label gap, checkbox-to-text gap |
| sm | 8px | `p-2` / `gap-2` | Input icon left padding, inline element gap |
| md | 16px | `p-4` / `gap-4` | Default horizontal screen padding, field spacing |
| lg | 24px | `p-6` / `gap-6` | Section vertical gap, button top margin |
| xl | 32px | `p-8` / `gap-8` | Hero illustration top offset |
| 2xl | 48px | `p-12` / `gap-12` | Onboarding logo vertical centering offset |
| 3xl | 64px | `p-16` / `gap-16` | Not used in auth screens |

**Screen-level padding:** `px-6` (24px) on all auth and onboarding screens.

**Exceptions:**
- Touch targets for all tappable elements: minimum 44px height (Apple HIG requirement) — use `min-h-[44px]` on pressable elements
- Primary button height: 52px (`h-[52px]`) — matches Figma button height
- Input field height: 52px (`h-[52px]`) — visual parity with button
- Dot indicator (onboarding pagination): 8px diameter active, 6px inactive — `w-2 h-2` active, `w-1.5 h-1.5` inactive

## Typography

System font stack only (no custom font loading in Phase 2). All sizes in sp/px units, applied via NativeWind `text-*` classes.

| Role | Size | NativeWind Class | Weight | Weight Class | Line Height | Usage |
|------|------|------------------|--------|--------------|-------------|-------|
| Display | 28px | `text-[28px]` | Bold (700) | `font-bold` | 1.2 | Onboarding headings ("Chào mừng đến với Ủ", "Bắt đầu hành trình khỏe mạnh") |
| Heading | 24px | `text-2xl` | Bold (700) | `font-bold` | 1.25 | Screen titles ("Đăng nhập", "Tạo tài khoản", "Hoàn thiện hồ sơ") |
| Body | 16px | `text-base` | Regular (400) | `font-normal` | 1.5 | Form labels, input placeholder text, descriptive subtitles |
| Label | 14px | `text-sm` | Regular (400) | `font-normal` | 1.4 | Field validation errors, helper text, "Quên mật khẩu?", Terms inline text |

**Typography rules:**
- Maximum 2 weights in use across Phase 2: Regular (400) and Bold (700).
- No italic text.
- No text smaller than 14px anywhere (accessibility floor).
- Vietnamese diacritical marks must render correctly on system font — do not use fonts with incomplete Unicode coverage.

- 60%: White screen backgrounds (`#FFFFFF`)
- 30%: Light grey surfaces for inputs + safe area (`#F5F5F5`)

## Interaction & Animation Contract

| Interaction | Behavior | Implementation |
|-------------|----------|----------------|
| Onboarding screen transition | Horizontal slide — next screen pushes in from right | Expo Router Stack default animation |
| Onboarding back swipe | Disabled on all 3 onboarding screens — use `gestureEnabled: false` in Stack screen options | Stack screenOptions |
| Button tap feedback | Scale down to 0.97 on press, restore on release | `react-native-reanimated` `useAnimatedStyle` or `Pressable` with `transform` |
| Input focus | Border color transitions to green (`border-primary`), border width 2px | Controlled via `onFocus` / `onBlur` state |
| Password visibility toggle | Eye icon toggles — no animation, instant show/hide | `secureTextEntry` state toggle |
| Keyboard avoidance | `KeyboardAvoidingView` wrapping `ScrollView` on all auth forms | `behavior="padding"` on iOS, `behavior="height"` on Android |
| Button loading state | Replace button label with white `ActivityIndicator` (size "small") — button remains disabled | Conditional render |
| Error message appearance | Slide down from above + fade in (100ms) | `react-native-reanimated` `FadeInDown` entering animation |
| OAuth buttons | Google: opens browser via native SDK. Apple: native system sheet. Both: disable button + show spinner during flow | SDK-handled |
| Session expired toast | `expo-toast` or custom toast component at top of screen — "Phiên đăng nhập hết hạn. Vui lòng đăng nhập lại." — green check → red warning. Duration 4000ms | Custom toast via Zustand notification state |
| Profile goal selection | Selected card animates border to green (150ms spring) | `Animated.spring` on border color |
| Complete profile — submit disabled | Button `opacity-50` while any field empty — re-evaluates on every keystroke | Derived from form state |

## State Diagram

```
App Cold Start
    │
    ├─ onboarding_seen = false ──► Onboarding Screen 1 → 2 → 3
    │                                      │              │
    │                              "Tạo tài khoản"  "Đăng nhập"
    │                                      │              │
    │                              /(auth)/register  /(auth)/login
    │
    └─ onboarding_seen = true
           │
           ├─ isAuthenticated = false ──► /(auth)/login
           │
           └─ isAuthenticated = true
                   │
                   ├─ profileCompleted = false ──► /(auth)/complete-profile
                   │
                   └─ profileCompleted = true ──► /(tabs) (main app)

Auth Actions:
  Register (email) → complete-profile → /(tabs)
  Login (email) → complete-profile (if needed) OR /(tabs)
  Google OAuth → complete-profile (if new user) OR /(tabs) (returning)
  Apple Sign In → complete-profile (if new user) OR /(tabs) (returning)
  Logout → clear tokens → /(auth)/login (onboarding_seen preserved)
  Session expired → /(auth)/login + toast
```
