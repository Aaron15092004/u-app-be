| Boundary | Description |
|----------|-------------|
| Admin/export -> backend | Raw redeem codes exist transiently during generation/export only. |
| Mobile/manual input -> backend | User-supplied code strings cross into normalization and hash lookup. |
| Package registry -> repo | New npm packages enter backend/mobile dependency tree. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01-01 | Information Disclosure | `redeem-code.service.ts` | mitigate | No persisted raw-code API; only HMAC hashes and HTTPS payload helper for generation/export responses per D-01/D-02. |
| T-01-01-02 | Tampering | Code normalization | mitigate | Normalize spaces, hyphens, and case before hashing so printed/manual variants map to one lookup. |
| T-01-01-03 | Denial of Service | Entitlement quota metadata | mitigate | Use finite high daily quota constants per D-04; do not expose unlimited/no-limit flags. |
| T-01-01-SC | Tampering | npm installs | mitigate | Use packages approved in `01-RESEARCH.md` Package Legitimacy Audit; do not add unaudited packages. |
</threat_model>
