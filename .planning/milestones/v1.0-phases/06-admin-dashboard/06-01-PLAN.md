**Phase:** 6 — Admin Web Dashboard
**Wave:** 1

---

## Goal

This plan makes the three schema/middleware changes that every subsequent plan depends on. It MUST be deployed in exact order: schema changes first, then middleware changes, then seed. The isActive middleware change is a breaking change — if deployed before the schema migration, every existing user gets a 401 because their document has no isActive field. Deploy order is non-negotiable.

**Deploy order:**
1. Deploy schema changes (User.isActive + FoodItem.imageUrl) — MongoDB Atlas automatically treats missing fields as `undefined`; the `default: true` in Mongoose means `findById` returns `isActive: undefined` for existing docs before re-save. To avoid a boot-time mass-lockout, the authenticate middleware uses `user.isActive !== false` (not `=== true`) — this allows `undefined` to pass.
2. Deploy middleware changes (authenticate isActive check + requireAdmin).
3. Run `npm run seed:admin` once.

### Task 3: Patch authenticate middleware — add isActive check; add requireAdmin

**File:** `backend/src/middleware/auth.middleware.ts`
**Action:** Edit

**Change 1 — authenticate function:** After `verifyAccessToken(token)` succeeds, do a DB lookup to check `isActive`. The lookup must be async; change the function signature to `async`. Use `User.findById(payload.sub).select('isActive')` and check `user.isActive !== false` (not `=== true`) — this safely handles existing docs where isActive was not set yet, treating undefined as active.

The updated authenticate function:
```typescript
export async function authenticate(req: Request, res: Response, next: NextFunction): Promise<void> {
  const authHeader = req.headers.authorization;

  if (!authHeader || !authHeader.startsWith('Bearer ')) {
    error(res, 'Token không hợp lệ hoặc đã hết hạn', 401);
    return;
  }

  const token = authHeader.slice(7);

  try {
    const payload = verifyAccessToken(token);
    const user = await User.findById(payload.sub).select('isActive');
    if (!user || user.isActive === false) {
      error(res, 'Tài khoản đã bị khóa', 401);
      return;
    }
    (req as AuthRequest).user = { id: payload.sub, role: payload.role };
    next();
  } catch {
    error(res, 'Token không hợp lệ hoặc đã hết hạn', 401);
  }
}
```

**Change 2 — add requireAdmin:** Add a new exported function after `requireProfile`:
```typescript
export function requireAdmin(req: Request, res: Response, next: NextFunction): void {
  const user = (req as AuthRequest).user;
  if (!user || user.role !== 'admin') {
    error(res, 'Chỉ quản trị viên mới có quyền truy cập', 403);
    return;
  }
  next();
}
```

The `AuthRequest` interface and imports do not change.

### Task 4: Create seed:admin script

**File:** `backend/src/scripts/seed-admin.ts`

```typescript
import 'dotenv/config';
import mongoose from 'mongoose';
import bcrypt from 'bcryptjs';
import User from '../models/User';

async function seed(): Promise<void> {
  const email = process.env.ADMIN_EMAIL;
  const password = process.env.ADMIN_PASSWORD;

  if (!email || !password) {
    throw new Error('ADMIN_EMAIL and ADMIN_PASSWORD must be set in environment');
  }
  if (password.length < 8) {
    throw new Error('ADMIN_PASSWORD must be at least 8 characters');
  }

  if (!process.env.MONGODB_URI) throw new Error('MONGODB_URI is not set');
  await mongoose.connect(process.env.MONGODB_URI);
  console.log('Connected to MongoDB');

  const existing = await User.findOne({ email: email.toLowerCase() });
  if (existing) {
    console.log(`Admin account already exists: ${email}. Skipping.`);
    await mongoose.disconnect();
    return;
  }

  const passwordHash = await bcrypt.hash(password, 10);
  await User.create({
    email: email.toLowerCase(),
    passwordHash,
    name: 'Admin',
    role: 'admin',
    isActive: true,
    profileCompleted: true,
  });

  console.log(`Admin account created: ${email}`);
  await mongoose.disconnect();
}

seed().catch(err => {
  console.error(err);
  mongoose.disconnect().finally(() => process.exit(1));
});
```
