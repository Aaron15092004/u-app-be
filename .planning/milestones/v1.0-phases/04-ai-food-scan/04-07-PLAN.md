## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-07-01 | Injection | search.tsx — user query to GET /api/food/items?q= | mitigate | Backend validates q via searchItemsSchema (min 2 / max 100 chars); $text operator (not raw regex); debounce 300ms reduces request rate |
| T-04-07-02 | Information Disclosure | diary.tsx — DELETE /api/food/logs/:id IDOR | mitigate | Backend deleteFoodLog service filters by both _id AND userId (from JWT); client can only supply :id — userId not controllable from client |
| T-04-07-03 | Spoofing | (tabs)/index.tsx D-69 buttons — navigation entry point | accept | D-69 explicitly specifies temporary buttons; no auth bypass; buttons navigate to authenticated stack routes |
| T-04-07-04 | Tampering | ServingSizeSheet — client calculates scaled nutrition | accept | Nutrition scaling is arithmetic from seed data (kcalPer100g * grams / 100); no server-side validation needed for MVP; values are estimates not medical data |
