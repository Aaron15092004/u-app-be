    FILE 2: Create backend/src/api/food/food.service.ts

    Imports: mongoose, FoodLog from models/FoodLog, FoodItem from models/FoodItem, vietnamDayStart from utils/date.

    Error helper (same pattern as bmi.service.ts): function makeError(message, statusCode): Error & { statusCode }.

    checkScanRateLimit(userId: string): Promise<boolean>
    - todayStart = vietnamDayStart(new Date())
    - tomorrowStart = new Date(todayStart.getTime() + 86400000)
    - count = await FoodLog.countDocuments({ userId: new mongoose.Types.ObjectId(userId), date: { $gte: todayStart, $lt: tomorrowStart }, aiProvider: { $ne: 'manual' } })
    - return count >= 20

    saveFoodLog(userId: string, body: z.infer<typeof saveFoodLogSchema>): Promise<object>
    - doc = await FoodLog.create({ userId: new mongoose.Types.ObjectId(userId), date: new Date(), ...body, imageUrl: body.imageUrl ?? null })
    - return doc.toObject()

    getFoodLogsForDate(userId: string, dateStr: string): Promise<object[]>
    - dayStart = vietnamDayStart(new Date(dateStr))
    - dayEnd = new Date(dayStart.getTime() + 86400000)
    - return FoodLog.find({ userId: new mongoose.Types.ObjectId(userId), date: { $gte: dayStart, $lt: dayEnd } }).sort({ createdAt: -1 }).lean()

    deleteFoodLog(userId: string, logId: string): Promise<void>
    - result = await FoodLog.deleteOne({ _id: new mongoose.Types.ObjectId(logId), userId: new mongoose.Types.ObjectId(userId) })
    - if result.deletedCount === 0, throw makeError('Không tìm thấy bữa ăn', 404)

    searchFoodItems(query: string): Promise<object[]>
    - if (!query || query.trim().length < 2) return []
    - return FoodItem.find({ $text: { $search: query.trim() } }, { score: { $meta: 'textScore' } }).sort({ score: { $meta: 'textScore' } }).limit(10).lean()
  </action>

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-04-03-01 | Elevation of Privilege | All food endpoints | mitigate | authenticate middleware required on all 5 routes; JWT verify in middleware rejects invalid/expired tokens with 401 |
| T-04-03-02 | Denial of Service | POST /api/food/scan — OpenAI spend + latency | mitigate | D-72: checkScanRateLimit before analyzeImage call; 20 scans/user/day server-side enforced; multer 5MB file limit already in upload.middleware.ts |
| T-04-03-03 | Tampering | POST /api/food/logs — client supplies nutrition values | accept | Nutrition values from AI result screen are truthy; Zod validation prevents malformed types; no server-side recalculation needed for MVP |
| T-04-03-04 | Information Disclosure | GET /api/food/logs + DELETE — IDOR | mitigate | All queries include `userId` from JWT (not from request body); deleteFoodLog passes both _id AND userId to deleteOne — cross-user access impossible |
| T-04-03-05 | Injection | GET /api/food/items?q= — search input | mitigate | $text operator (not regex) used; searchItemsSchema enforces min 2 / max 100 chars; no raw user string concatenated into query |
| T-04-03-06 | Denial of Service | File upload abuse — large files | mitigate | multer already configured with 5MB limit in upload.middleware.ts; no additional config needed |
