<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| mobile client ŌåÆ /api/auth/* | Untrusted JSON bodies (email, password, refresh tokens) cross here |
| /api/auth/* ŌåÆ MongoDB | User documents persisted with password hash + token hashes |
| backend ŌåÆ Resend API | Outbound HTTPS with API key, contains user email address |
| backend ŌåÆ JWT secrets (env) | Sign/verify happen in process memory; secrets must never log |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | JWT access token | mitigate | sign with `process.env.JWT_ACCESS_SECRET`, HS256 default; verify before trusting payload (D-22) |
| T-02-02 | Information Disclosure | Refresh token at rest | mitigate | store only bcrypt hash via `hashToken` in `User.refreshTokenHash` (D-22); raw token never persisted |
| T-02-03 | Information Disclosure | Password reset token at rest | mitigate | store only bcrypt hash via `hashToken` in `User.passwordResetTokenHash` with 1-hour `passwordResetTokenExpiry` (D-27) |
| T-02-04 | Tampering | JWT_ACCESS_SECRET / JWT_REFRESH_SECRET | mitigate | env-only, fail-fast on missing secret at module init; never log; .env.example uses placeholder values |
| T-02-05 | Information Disclosure | Password in DB | mitigate | bcrypt cost 10 via `hashPassword`; never log plain password; controllers strip password from any response payload |
| T-02-06 | Spoofing | Missing Authorization header | mitigate | `authenticate` middleware returns 401 before any handler executes |
| T-02-07 | Elevation of Privilege | requireProfile bypass | mitigate | `requireProfile` requires `profileCompleted=true`; downstream routes mount middleware in correct order (D-33) |
| T-02-08 | Repudiation | Outbound Resend email | accept | Resend dashboard logs all sends; no PII in subject; user can be told to check spam |
| T-02-SC | Tampering | npm install of `resend` | mitigate | Package `resend` is the official Resend Node SDK (publisher: resend) ŌĆö verified during planning; legitimate package on npmjs.com; no blocking checkpoint required as the package is well-established (>500k weekly downloads, official vendor SDK) |
</threat_model>
